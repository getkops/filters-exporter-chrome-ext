# Firefox Add-ons (AMO) Publishing Guide

> Step-by-step guide to publish the Kops Filter Exporter extension on
> [addons.mozilla.org](https://addons.mozilla.org) (AMO).
>
> See `CHROME_STORE_GUIDE.md` for the Chrome Web Store. The listing copy
> (name, description, screenshots) is shared between the two stores.

---

## Pre-requisites

- ✅ Extension code ready and tested
- ✅ Privacy policy created (`PRIVACY_POLICY.md`)
- ✅ Icons available (128×128)
- [ ] A Mozilla (Firefox) account — the AMO developer account is **free** (no fee, unlike Chrome's $5)
- [ ] Screenshots (reuse the 1280×800 ones from the Chrome listing)
- [ ] Firefox **128 or newer** to test (the extension requires 128+ — see [Minimum version](#appendix-a--minimum-firefox-version))

---

## Step 1 — Create a Developer Account

1. Go to the **[Add-on Developer Hub](https://addons.mozilla.org/developers/)**
2. Sign in with a Mozilla account (use your Kops business account)
3. Accept the Firefox Add-on Distribution Agreement — **there is no registration fee**

---

## Step 2 — Build the Firefox ZIP

The Firefox package is **not** the same zip as Chrome: Firefox needs a `background.scripts`
event page (not a service worker) and a Gecko add-on id. Those differences are generated
automatically from the Chrome `manifest.json` by `scripts/build-firefox.mjs` — you never edit
a second manifest by hand.

**Option A — from a tagged release (recommended):**
Download `kops-filter-exporter-firefox-v*.zip` from the
[Releases page](https://github.com/getkops/filters-exporter-chrome-ext/releases/latest)
(the release workflow builds it alongside the Chrome zip).

**Option B — locally:**
```bash
mise install                # pinned Node 24.17.0 + pnpm 11.8.0
pnpm install
pnpm build:firefox          # builds dist/ then stages build/firefox/
pnpm lint:firefox           # web-ext lint — expect 0 errors (see Appendix B)
cd build/firefox && zip -r ../../kops-filter-exporter-firefox.zip . -x "*.git*"
```

> ⚠️ The zip must contain `manifest.json` at the **root** level (the `build/firefox/` layout
> already guarantees this). The Gecko add-on id is `kops-filter-exporter@getkops.com` — this is
> **permanent** once the listing exists; never change it.

## Step 4 — Submit the Add-on

1. Developer Hub → **Submit a New Add-on**
2. Distribution channel → **"On this site"** (listed — public on AMO). *(Not "On your own", which is unlisted/self-hosted.)*
3. Upload `kops-filter-exporter-firefox-v*.zip`
4. Automated validation runs (the same `addons-linter` that `pnpm lint:firefox` uses). It should pass with **0 errors** — the warnings are benign and explained in [Appendix B](#appendix-b--web-ext-lint-warnings-all-benign).
5. **Source code submission (required):** because the code is bundled with esbuild, AMO asks whether you use a build step — answer **yes** and provide the sources. The repo is public (MIT), so either link the tagged commit or upload a source archive:
   ```bash
   git archive --format=zip -o kops-filter-exporter-src-v2.1.1.zip v2.1.1
   ```
   Paste the build instructions from [Appendix C](#appendix-c--notes-to-reviewer-ready-to-paste) into the reviewer-notes field.

## Step 6 — Data Collection Disclosure

This add-on collects **no data**. This is declared two ways, which must agree:

- In the manifest: `browser_specific_settings.gecko.data_collection_permissions = { "required": ["none"] }` (generated by `build-firefox.mjs`).
- In the AMO submission form, if prompted: select **"No"** / "does not collect data".

This is consistent with `PRIVACY_POLICY.md` (no collection, no transmission, no analytics).

---

## Step 7 — Submit for Review

1. Confirm all listing fields, screenshots, and the privacy policy
2. Click **Submit Version**
3. Listed add-ons go through automated validation plus possible human review. Turnaround ranges from a few hours to a few days.
4. On approval, **Mozilla signs the add-on automatically** and it goes live on AMO — no separate signing step is needed.

---

## After Publishing

Future updates:

1. Bump `version` in `manifest.json` (the Firefox manifest inherits it automatically)
2. Tag the release (`vX.Y.Z`) → CI attaches both the Chrome and Firefox zips to the GitHub Release
3. Developer Hub → the add-on → **Upload New Version** → upload the new `…-firefox-…zip`
4. Re-submit the source archive + reviewer notes (Appendix C) if the build changed

> Tip: once the AMO API keys are set up (JWT issuer + secret), the manual upload can later be
> replaced by a `web-ext sign --channel=listed` step in `release.yml`. Deferred for now.

## Appendix A — Minimum Firefox version

`strict_min_version` is **128.0**. The interception in `src/inject.ts` runs as a
`"world": "MAIN"` content script, and MAIN-world content scripts are only supported from
**Firefox 128** onward. Firefox 128 is an ESR release, so this covers all current stable and ESR
users. Do not lower it below 128 — interception (the core feature) would silently break.

## Appendix B — `web-ext lint` warnings (all benign)

`pnpm lint:firefox` reports **0 errors, 0 notices, 3 warnings**. None block submission:

| Warning | Why it's safe |
| --- | --- |
| `UNSAFE_VAR_ASSIGNMENT` (`dist/popup/popup.js`) | The only `innerHTML` writes are inside the bundled **Preact** runtime. Our own source has **zero** `innerHTML` / `dangerouslySetInnerHTML` uses (`grep innerHTML src/` → none), so no untrusted HTML is ever assigned. |
| `KEY_FIREFOX_UNSUPPORTED_BY_MIN_VERSION` (`data_collection_permissions`) | The data-collection key is enforced in-browser only on newer Firefox, but we intentionally keep `strict_min_version: 128` for MAIN-world reach. AMO still reads the key for the listing, and Firefox 128–139 simply ignores it. |
| `KEY_FIREFOX_ANDROID_UNSUPPORTED_BY_MIN_VERSION` | Same as above, for the Firefox-for-Android target. |

## Appendix C — Notes to reviewer (ready to paste)

```
This add-on is open-source (MIT): https://github.com/getkops/filters-exporter-chrome-ext

BUILD (reproduces the submitted files exactly):
  Toolchain is pinned with mise (see mise.toml): Node 24.17.0, pnpm 11.8.0.
    mise install
    pnpm install --frozen-lockfile
    pnpm build:firefox
  Output is build/firefox/ (manifest.json + dist/ + _locales/ + icons/ + LICENSE),
  which is exactly what was zipped and uploaded.

WHAT IT DOES:
  Content scripts on v-tools.com and souk.to intercept the user's OWN filter/alert
  API responses (fetch/XHR monkey-patching in the page's MAIN world), normalize them
  into a typed JSON envelope, and store it in local storage. The popup lets the user
  download that JSON. Nothing is transmitted to any server; there is no analytics,
  no tracking, and no data collection (data_collection_permissions: ["none"]).

BUNDLED CODE:
  dist/*.js are esbuild bundles (not minified) of the TypeScript in src/. The popup UI
  is built with Preact (bundled into dist/popup/popup.js) — this is the source of the
  single UNSAFE_VAR_ASSIGNMENT/innerHTML warning; our own code never assigns innerHTML.
```
